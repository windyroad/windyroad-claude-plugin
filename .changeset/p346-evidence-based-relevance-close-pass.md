P346 / ADR-079: Evidence-based relevance-close pass for the problem backlog (Phase 1: file-no-longer-exists).

`/wr-itil:review-problems` SKILL.md gains a new **Step 4.6 — Relevance-close pass** between Step 4.5 (Inbound-discovery) and Step 5 (README rewrite) that closes the structural outflow gap in the backlog. The pass iterates over `.open.md` / `.known-error.md` tickets aged ≥ 7 days, invokes the canonical `packages/itil/scripts/evaluate-relevance.sh` body via the ADR-049 PATH shim `wr-itil-evaluate-relevance`, and auto-closes tickets whose referenced file paths no longer exist in the codebase. Phase 1 ships the single highest-confidence evidence shape — "file no longer exists in `git ls-files`" — closest analog to P334/P336 close-on-evidence patterns. Subsequent evidence shapes (ADR-supersession, duplicate-of-X, "concern no longer concerning", SKILL-contract-superseded) are deferred to sibling tickets per ADR-079 Phase 1 scope discipline.

**User direction (verbatim, 2026-05-31)**: *"Ok, I'm happy for a skill executed as part of review problems that closes tickets that are no longer relevant, but not just because they are old"* — the relevance signal MUST be observable per ADR-026; age is a **gating** condition (don't bother evaluating fresh tickets), never the **closing** condition.

**Mechanism**:
- The canonical script extracts file-path candidates from each ticket body matching `(packages|docs|.changeset|src|test|scripts)/[A-Za-z0-9._/-]+\.(md|sh|ts|tsx|js|jsx|json|yml|yaml|bats|py|txt|html)`, drops self-references to `docs/problems/*`, and runs `git ls-files --error-unmatch` on each. Emits `CLOSE-CANDIDATE` (exit 0) only when ALL extracted paths return zero AND at least one was extracted; otherwise `KEEP` (exit 1) or `SKIP` (exit 2 — age gate, no Reported date, no extractable paths).
- Auto-close action: Edit a `## Closed as no longer relevant` audit section into the ticket body (cite + persist + uncertainty per ADR-026 — paths checked, closed-on date, script body re-runnable, reversibility via `git revert`), then `git mv` Open|Known Error → Closed bypassing Verifying (lifecycle extension per ADR-079; no fix was released so no verification phase applies).
- Commit grain: all relevance-closes from one review pass batch into ONE commit `chore(problems): relevance-close pass — close N tickets as no longer relevant` per ADR-014 single-commit grain, mirroring `/wr-itil:transition-problems` P139 batch pattern. Step 5's README refresh rides the same commit.
- AFK-policy-authorised silent proceed per ADR-013 Rule 5 + ADR-044 category 4 silent framework action (file existence is empirical, not user judgment).

**Lifecycle extension (ADR-022 extension, not modification)**: `/wr-itil:manage-problem` SKILL.md lifecycle table's Closed row updated to reflect the new alternative entry path. ADR-022 status-transition mechanics for Open / Known Error / Verifying / Closed remain unchanged; the extension precedent mirrors ADR-026 line 109's `ADR-022 extension (not modification)` shape.

**Behavioural second-source**: `packages/itil/scripts/test/evaluate-relevance.bats` ships 18 fixtures covering: script existence + executability + shim dispatch; usage / error path (no args / missing file → exit 3); age gate (fresh ticket / no Reported date → SKIP exit 2); no extractable paths (empty body / only self-references → SKIP exit 2); CLOSE-CANDIDATE (all-absent / single-absent / known-error-state — exit 0); KEEP (all-present / mixed present — exit 1); custom age gate; verdict-shape output contract (CLOSE-CANDIDATE / KEEP / SKIP first-line keyword). 18/18 GREEN. Real-backlog smoke test (2026-05-31, 143 open/known-error tickets): 6 CLOSE-CANDIDATE (4.2%), 44 KEEP, 93 SKIP — conservative behaviour confirmed; no false-positive closes on tickets with live file references.

Architect verdict 2026-05-31: ALIGNED-WITH-NITS (must-do nit: explicit ADR-022 extension cite — done in ADR-079; minor nit: do not auto-stamp `human-oversight: confirmed` — honoured; ADR-079 remains `proposed` for orchestrator-level drain ratification per ADR-066 line 50).
