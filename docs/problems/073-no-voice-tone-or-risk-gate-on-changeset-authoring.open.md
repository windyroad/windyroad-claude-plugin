## Description

`.changeset/*.md` bodies populate **four external-visible surfaces** downstream of authoring:

1. **`packages/<name>/CHANGELOG.md`** — generated by `changeset version` and committed to `main`. Public on GitHub; shipped inside every published npm package.
2. **Release PR body** (`changeset-release/main` branch) — the aggregate of pending changeset summaries appears on the auto-created Release PR; publicly visible on GitHub while the PR is open.
3. **GitHub Release body** — the tagged-release page on `github.com/windyroad/agent-plugins/releases` carries the changelog entry verbatim.
4. **npm metadata** — the changelog entry is fetched by tooling (e.g. `npx changeset:check`, `npm-check-updates`) and appears in package-registry UI scrapers.

Despite this, **the existing voice-tone and risk-scoring gate designs (P038, P064) do not include `.changeset/*.md` in their `PreToolUse` surface inventory**. Both tickets enumerate `gh issue create`, `gh issue comment`, `gh pr create`, `gh pr comment`, `gh api .../security-advisories`, `gh api .../comments`, and `npm publish` (with README diff) — but none mention changeset authoring. By the time `npm publish` fires (P064's last-moment leak-diff catch), the CHANGELOG.md has already been committed and the Release PR has already been public for as long as the PR was open. Gating at publish is too late; the content is already external.

Observed this session (2026-04-20, AFK iter 6): three changesets authored (`.changeset/p062-*.md`, `.changeset/p063-*.md`, `.changeset/p068-*.md`), each with multi-paragraph prose describing architectural changes. Two of them shipped in `@windyroad/itil@0.9.0` / `@windyroad/retrospective@0.4.0` before any voice-tone or risk check ran over the body text. No gate fired. No leak was observed (manually reviewed on stage), but the audit trail did not record a check.

## Symptoms

- Changeset body prose reaches CHANGELOG.md / Release PR / GitHub Release / npm with no recorded voice-tone or risk-gate verdict.
- `PreToolUse:Write` hook patterns cover source-code files and docs but NOT `.changeset/*.md` (verified: `packages/risk-scorer/hooks/git-push-gate.sh` intercepts `gh pr merge`; no hook watches `.changeset/`).
- P038 (voice-tone gate) and P064 (risk-scoring gate) surface inventories enumerate `gh issue`, `gh pr`, `npm publish` — never `.changeset/`.
- The Release PR body is aggregated from pending changesets by the `changesets/action` GitHub Action; no content check runs on the aggregate either.
- An agent authoring a changeset with AI-tell patterns ("I've implemented...", "I've updated..."), client-specific language, or a leak of an unannounced feature/client name can ship the text unchecked.

## Workaround

Manual review of each changeset before commit. Same "manually police AI output" pain P038 calls out for `gh issue comment` applies here verbatim.

## Root Cause Analysis

### Structural

P038 (voice-tone gate) and P064 (risk gate) were designed around the `gh` / `npm publish` CLI surface — the outbound tool calls Claude typically makes when submitting issues, PRs, or published packages. Changeset files are authored via `Write` / `Edit` tool calls against `.changeset/<name>.md`, not via `gh` / `npm`. The surface inventory conflated "how external comms reach an audience" (the `gh` / `npm` call) with "when the content exists" (the file write). Changeset bodies exist **at file-write time**, publish only **renders them to CHANGELOG.md**; gating at publish misses the author-time intervention window.

### Fix strategy

Extend the gate surface inventory in both P038 and P064 to include:

- **`PreToolUse:Write` / `PreToolUse:Edit` on paths matching `.changeset/*.md`**
  - Trigger the same voice-tone classifier that P038's gh-comment path will use (reject AI-tell patterns, apply the voice profile, halt-and-prompt the agent for a rewrite).
  - Trigger the same risk-leak-pattern rules that P064's gh-comment / npm-publish paths will use (reject client names, unannounced features, pricing/revenue figures, user counts).
  - AFK non-interactive branch (per ADR-013 Rule 6): halt the write with the detected issues surfaced; do not auto-rewrite.

- **Additionally** consider a `PreToolUse:Bash` on `npx changeset add` (the CLI path that prompts for a changeset body interactively) — the hook cannot reach inside the interactive prompt, but firing an advisory notice at invocation time reminds the author that the body will be gated when written.

No new standalone ADR is proposed — this extends the surface inventories that P038 and P064 (or their ADRs when drafted) already govern. The ticket exists to make the gap visible and to ensure neither P038 nor P064 ships without the changeset row on its surface list.
