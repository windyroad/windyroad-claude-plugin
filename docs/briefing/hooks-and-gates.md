# Hooks and Gates

Learnings about PreToolUse/PostToolUse gate behaviour, markers, and expiry.

## What You Need to Know

- **Four edit gates fire on every edit**: architect review, JTBD review, WIP risk assessment, and TDD enforcement. Each requires its own agent delegation before the edit is allowed. Plan for this overhead. <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-04-19 -->
- **Risk-scorer agents are tool-restricted to `Read + Glob`** — no `AskUserQuestion`, no `EnterPlanMode` (sub-agents via Task can't enter plan mode either). Any "scorer asks the user" design must split scorer/orchestrator: scorer emits structured markers, the calling skill/primary owns the interaction (P021). <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-04-19 -->
- **Rotated historical entries live in two siblings**: [`hooks-and-gates-archive.md`](./hooks-and-gates-archive.md) (2026-05-04/05) and [`hooks-and-gates-archive-pre-2026-05-04.md`](./hooks-and-gates-archive-pre-2026-05-04.md) (2026-04-19→04-26). Load when investigating release-pipeline / changeset / `.html`-write friction. <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-05-13 -->
- **`BYPASS_*_GATE=1` does not reach a PreToolUse hook from the same Bash call** (P173 verifying) — inline and `export; git commit` both fail since the hook runs before git. Working pattern: write a wrapper script (`export BYPASS_*_GATE=1; git commit -F msg`) and invoke `bash /tmp/commit-<ticket>.sh`; the hook fires on the outer `bash ...` text and the export is visible to the child shell. Use only when bypass conditions are legitimately met. <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-05-16 -->
- **Create-gate marker races between concurrent sessions** (P119 verifying; P260) — the runtime-sid file is per-machine not per-process, so orchestrator turn + iter subprocess clobber it (last-writer-wins) → marker under the wrong SID. Workaround: spam-write the marker under recent `/tmp/<system>-announced-*` UUIDs + the runtime-sid value. ADR-050 amended-in-place 2026-05-26 (Option C = bounded multi-UUID marker-write); impl P260. <!-- signal-score: +2 | last-classified: 2026-05-26 | first-written: 2026-05-18 -->
- **The README-refresh hook (P165) honours registered `RISK_BYPASS:` trailers** (P262/P265 shipped, verifying) — it would deny a bare `docs(problems): capture` commit (new ticket = ranking-bearing), but `/wr-itil:capture-problem` Step 6 emits the `capture-deferred-readme` trailer (second `-m` para) which clears the README gate ONLY (still risk-scored — don't stage README on a capture). Bypass tokens live in `_README_REFRESH_BYPASS_TRAILERS` (`adr-031-migration` also clears `risk-score-commit-gate.sh`; `capture-deferred-readme` is README-gate-only); new tokens go in that array AND the ADR-014 table — the two gates carry independent recognition lists. <!-- signal-score: +2 | last-classified: 2026-05-26 | first-written: 2026-05-18 -->
- **`/wr-itil:capture-problem` Step 2 dedup listing truncates at ~top-10** (P252 verifying) — `match_count` is total but the agent inspects only the first ~10, so a duplicate beyond that is missed. When `match_count > 10`, `grep -liE '<kw1>|<kw2>|<kw3>'` all matches, or use `/wr-itil:manage-problem` (wider dedup). <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-05-18 -->
- **Multi-decision-file changes DEADLOCK the architect edit-gate** (P303 open) — verdict-grep reads "ISSUES FOUND" as FAIL, the gate re-locks on every `docs/decisions/` hash drift, post-approval re-reviews fail (work not on disk yet). Recover after a genuine approval (user-authorised): assert `/tmp/architect-reviewed-$SID` + `rm` the `.hash` sibling. The same verdict-grep false-FAIL (P181/P217) fires on a SINGLE-file edit when an APPROVED re-review's prose merely contains the substring `ISSUES FOUND` — same marker-assert recovery. (P261 hit this 2026-05-26.) <!-- signal-score: 2 | last-classified: 2026-05-26 | first-written: 2026-05-25 -->
- **A commit that removes/narrows a load-bearing currency hook is blocked by the cached OLD hook, and `BYPASS_*=1` can't reach it (P173 verifying) — land it from an EXTERNAL terminal.** The cached hook lags source so the superseded rule still fires, and the PreToolUse gate only intercepts in-session commits. Write the message to a repo-local real path (`.git/<name>.txt`); the session's `/tmp` is sandboxed and invisible to the user's terminal. <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-05-25 -->

## What Will Surprise You

- **Gate hooks once substring-matched `git commit` anywhere in a command body** — fired on grep/heredoc/sed/echo args (P268/P272 closed; P273/P274/P275 verifying). Fix: a `command_invokes_git_commit` leading-executable helper at `packages/shared/hooks/lib/command-detect.sh`. Until the cache catches up on the remaining hooks, avoid a literal `git commit` in argument vectors during a commit. <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-05-18 -->
